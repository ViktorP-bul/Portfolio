Design

Having gained some insight and knowledge on the [problem], target audiences and more, I used the acquired information and started redesigning the [solution]. The challenge was to try and implement what I had learned. My goal was to make sure that I took in as many gains and fixed as many pains I came across during my research. These gains/pains didn't only exist in the [existing product], but also in the other products I evaluated.
The first step into these redesigns, involved sketches in the form of wireframes. These were then shown to the development team and they picked the most suitable version based on what we learned from my research above. You can learn more about these wireframes here.
From there, I took the most suitable wireframe and turned it into a clickable prototype. My goal was to show the company owner and the development team how the game would look and play like if I added all the elements into the game without coding these changes. Another reasoning is that the wireframes were more about placement and this prototype brings everything together (including color, font and more). More about this phase can be read here.

Realisation and Evaluation

After tweaking the clickable prototype to make it abide by both the target audience as [client], the final step involved implementing the elements of the clickable prototype into the actual game. I worked together with the dev team to make sure that the look and feel mirrored the prototype AND to make sure the game was able to be played on the wall. More about this phase can be read here.
Unfortunately, I ran across several problems throughout this phase as well. First, was the unorganised coding of the [solution]. While the company has coding standards, this did not include the code within the [existing game]. This caused much confusion while trying to implement content into the game. To make sure I would still have the opportunity to learn about [learning goals], I was allowed to code [alternative product]. I then learned to code a [solution]. You can read more here​
Another [problem] arose. I did not have the opportunity to test the new version of the [existing game] with the target audience. I therefore could not prove that my designs were indeed "improved". That is why I wrote a recommendation document in case [client] wants to test the game in the future. More about this test can be read here.
